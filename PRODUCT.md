# Product

## Register

product

## Users

Fashion and e-commerce brand teams — photographers, marketing managers, and studio ops — who need to generate commercial product imagery at scale without a physical shoot. They work fast, have strong aesthetic opinions, and evaluate outputs against real campaign briefs.

## Product Purpose

Click is a virtual photography studio that uses AI to generate on-brand product shots from a configuration. Users set up a shoot once (category, style, mood, scene) and get a batch of usable images. Success looks like: images a brand would actually publish, generated in minutes, not days.

## Brand Personality

Precise. Studio-grade. Quietly capable.

The tool should feel like professional software — Figma-level craft applied to AI generation. Not a toy, not a generic SaaS dashboard. The UI should earn trust by looking like it knows what it's doing.

## Anti-references

- Generic AI tool dashboards (loud gradients, explainer copy everywhere, "magic wand" iconography)
- Numbered step wizards with eyebrow labels
- Heavy dark-mode glassmorphism with neon accents
- Canva-style UI (friendly but amateur)
- Midjourney's chaotic grid (no structure, no control)

## Design Principles

1. **Control without clutter** — Every field has a reason. No decorator inputs, no "coming soon" states masquerading as features.
2. **Vocabulary over chrome** — The UI communicates through its choices (color accents per shoot style, mood swatches) not through labels screaming at the user.
3. **Familiarity earns speed** — Inputs, buttons, and affordances should behave exactly as a seasoned tool user expects. No invented patterns.
4. **Output quality reflected in UI quality** — If the tool generates studio-grade imagery, the interface must look like it belongs next to Lightroom and Figma.
5. **Trust through restraint** — What's not there matters. No loading spinners in content, no modals as a first resort, no full-saturation accents on every element.

## Accessibility & Inclusion

WCAG AA minimum. Sufficient contrast on all interactive states. Keyboard navigable forms. `prefers-reduced-motion` respected.
